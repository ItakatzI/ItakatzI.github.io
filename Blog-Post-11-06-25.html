Behind the Scenes: Crafting My Personal Website with ChatGPT

June 11, 2025 • 8 min read

It all began on a quiet afternoon in Jerusalem, armed with nothing but a rough sketch on paper and a head full of ideas. I wanted a dashboard that wasn’t just another static page, but a living, breathing interface—something you’d expect from a polished SaaS platform, yet simple enough to host on GitHub Pages. My first drawing featured a narrow sidebar, crisp cards for each section, and a bold header that typed out my roles one by one. But as any developer knows, a sketch is just the starting line.

Before touching a single line of CSS, I spent hours refining the user flow. What should happen when someone clicks the “Stats” icon? How best to transition between “About” and “Experience” without a full page reload? I grabbed coffee, opened my notes app, and listed out every interaction: fade-ins, hover states, active icons. That plan became the backbone of our CSS and JavaScript strategy.

I decided on a mobile-first mindset—if it looked and felt smooth on my phone, it would shine on desktop. I tested color contrasts in the morning light of my home office and debated whether my accent color should be the vibrant blue of GitHub’s logos or a subtler gradient. Every choice mattered.

Enter ChatGPT: my tireless brainstorming partner. I’d paste in my sidebar HTML and ask, “How do I swap full labels for icons only, while keeping the active section highlighted?” Within seconds, I had a snippet that used data attributes and an .active class to toggle visibility. Next, I wondered about smooth fade-ins—ChatGPT suggested leveraging opacity transitions combined with display: none for clean toggles.

That back-and-forth felt less like machine chatter and more like pair programming. When I hit a snag—my skill badges weren’t centering perfectly—ChatGPT identified a missing justify-items: center rule in my grid layout. A single line of CSS, and suddenly my skills lined up like soldiers on parade.

Along the way, I picked up techniques I’ll carry forward to every project:

  • Structure with CSS variables: Defining --primary-color and --bg-light at the top made global theme changes a single search-and-replace.
  • Lazy-load interactive elements: By deferring the typing SVG until hover, the initial render stayed snappy and responsive.
  • Semantic HTML for accessibility: Ensuring each section had an aria-labelledby attribute meant screen readers could navigate seamlessly.
  • Minimal JavaScript: Rather than a bulky framework, a handful of query selectors and classList.toggle calls did the job with under 2 KB of code.

Here’s a taste of the JavaScript that powers section switching: 


// Grab all sidebar links and sections
const links = document.querySelectorAll('.nav-menu a');
const secs  = document.querySelectorAll('section');

links.forEach(link => {
  link.onclick = e => {
    e.preventDefault();
    // deactivate all
    links.forEach(l => l.parentNode.classList.remove('active'));
    secs.forEach(s => s.classList.remove('active'));
    // activate target
    link.parentNode.classList.add('active');
    document.getElementById(link.dataset.target).classList.add('active');
  };
});

Just seventy lines of code—and with the right CSS transitions, each section fades in and out like a scene change in a movie.

Design tweaks came in rapid succession. I restored hover animations on the main title, added subtle pulsing to emoji bullets, and ensured the entire content area filled the viewport gracefully. Under ChatGPT’s guidance, I even crafted a unicorn favicon in pure CSS for fun—because who doesn’t love a unicorn?

Testing across devices was the final step. My sister’s old tablet handled sidebar toggling smoothly, and my phone’s dark mode respected the gradients perfectly. A quick validation with Lighthouse showed a solid performance score, thanks to minimal dependencies and optimized images.

Reflecting on this journey, the biggest surprise wasn’t the code—it was how a conversational AI can amplify creativity. ChatGPT never gets tired, offers fresh ideas at odd hours, and reminds me of small best practices (like adding rel="noopener" on external links). It’s the ultimate co-author for modern web projects.

Building this unicorn-themed dashboard was a test of design, development, and collaboration. From rough sketches to polished interactions, every line of CSS and JavaScript was informed by a seamless chat.
