🐍 How I Built a Snake AI Showdown (And Accidentally Learned Game AI)

By Katzir • June 22, 2025

Hey there. If you’re a PM like me—fascinated by game AI, but slightly intimidated by all the mathy-sounding stuff—this is for you.

A while ago, I had this silly idea: “What if I built a Snake game that plays itself?” You know, like four different snakes, each with its own brain, battling it out on the screen. One moves randomly, one is aggressive, one tries to be clever… and one? One’s actually smart.

Turns out, that idea became the most fun I’ve ever had with JavaScript.

This post is half tutorial, half story. I’ll walk you through what I built, how it works, and what I learned along the way. If you’ve ever made a for loop and wondered what else you could do, you’re gonna love this.

🤖 What Is Game AI, Actually?

Before we dive in, let’s demystify “game AI.”

It’s not ChatGPT. It’s not Skynet. It’s basically: code that makes decisions instead of a player.

In Pac-Man, the ghosts have AI. In Super Mario, the Goombas walk at you like mindless zombies. In Tetris, the AI is you, panicking.

In Snake, the decision is super simple:

Where should the snake move next?

So our challenge is: how can we write different versions of that decision logic?

🎮 The Idea: Four AIs, One Arena

Here’s what I wanted:

  • A landing page with a big button: “Launch Simulation”
  • Four Snake games on the screen at once
  • Each AI playing Snake without human input
  • One AI per canvas:
  • 🟥 Random AI — basically a drunk snake
  • 🟧 Greedy AI — races to the apple, ignores consequences
  • 🟨 Tail-Aware AI — thinks before biting
  • 🟩 Smart AI — actually survives for a while

I wanted it hosted on GitHub Pages so I could show it off.

Spoiler: all of this is possible with just HTML, CSS, and JavaScript.

📦 The Setup

I made a folder:

snake-ai-showdown/
├── index.html
├── game.html
├── style.css
├── snake.js
└── README.md

Each AI has its own <canvas> and gets its own copy of the Snake game logic.

No libraries. No frameworks. Just vibes.

🧠 Let's Talk About the AIs

🟥 Random AI

const moves = getValidMoves(head);
return moves[Math.floor(Math.random() * moves.length)];

What does it do? Nothing useful. It just wiggles. It’s hilarious.

I watched it bonk into itself 3 seconds in and laughed like a maniac.

🟧 Greedy AI (A*)

Then I thought, “Can I make the snake actually try to get the apple?”

So I implemented A* — a pathfinding algorithm. Sounds fancy, but it’s like Google Maps with tunnel vision:

const path = aStar(snakeHead, apple);
return path[1]; // take the first step

It worked. The snake zooms straight to the apple. And then crashes into its own tail. Every time.

🟨 Tail-Aware AI

New plan: check if the path to the apple still allows tail access afterward.

if (isSafe(pathToApple)) return pathToApple[1];
else return pathToTail[1];

This blew my mind. It survives way longer. It slows down, loops, waits… it feels smart.

🟩 Smart AI

The boss AI combines:

  • Tail-aware logic
  • Fallbacks to tail-chasing
  • Loop detection
  • Restarting if trapped
if (safeToApple) return pathToApple[1];
else if (pathToTail) return pathToTail[1];
else return randomValidMove();

🔁 Loop Detection

Tail-Aware AI could pace in circles. So I added detection:

if (growthStalled && repeatingPositions) {{
  this.restart();
}}

It tracks the last 200 head positions and counts uniqueness. Too few = trapped.

🧪 What I Learned

  • AI doesn’t mean complicated. It means “decides things.”
  • A* is powerful but not perfect.
  • You can layer intelligence like Legos.
