🤖 Demystifying Game AI

Game AI in 2025 isn’t about sentient machines; it’s code that makes decisions instead of a player. In Pac-Man, ghosts decide where to go; in Mario, Goombas shuffle along. In Snake, the single question is:

Where should the snake move next?

We’ll implement four “brains” to answer that question differently.

🎮 Four AIs, One Arena

  • 🟥 Random AI — zero planning, pure chaos
  • 🟧 Greedy AI — A* to apple, ignores safety
  • 🟨 Tail-Aware AI — checks survival after apple
  • 🟩 Smart AI — adaptive, loop-proof, restarts

Hosted on GitHub Pages, all in pure HTML/CSS/JS—no frameworks required.

🧠 AI Deep Dive: Theory & Code

🟥 Random AI

Theory: A nondeterministic agent with no memory. Picks any legal move uniformly at random.

function randomValidMove() {
  const moves = getValidMoves(head);
  return moves[Math.floor(Math.random() * moves.length)];
}

Anecdote: I laughed so hard when it bonked itself immediately—I almost fell off my chair.

🟧 Greedy AI (A*)

Theory: A* pathfinding finds the shortest path using f(n)=g(n)+h(n). We use Manhattan distance for h.

function greedyMove() {
  const path = aStar(snakeHead, apple);
  return path ? path[1] : null;
}

Why it fails: It’s brilliant until it traps itself—like running blindly toward success and falling off a cliff.

🟨 Tail-Aware AI

Theory: Simulate “if I eat this apple, can I still reach my tail?”

function tailAwareMove() {
  const toApple = aStar(head, apple);
  if (toApple && isSafe(toApple)) {
    return toApple[1];
  }
  const toTail = aStar(head, tail);
  return toTail ? toTail[1] : randomValidMove();
}

isSafe(path) clones the snake, “grows” it along the path, then checks via A* whether head→tail remains reachable. My snake started planning better life decisions than I did in my 20s.

🟩 Smart AI

Theory: Layer behaviors with fallback and loop-escape:

  1. Go for apple if isSafe
  2. Else chase tail
  3. Else pick a random safe move
  4. If stuck (no growth + repeated head spots), restart()
function smartMove() {
  if (isSafe(pathToApple)) return pathToApple[1];
  if (pathToTail) return pathToTail[1];
  if (isStuckInLoop()) {
    restart();
    return;
  }
  return randomValidMove();
}

Loop detection: track last 200 head positions; if fewer than 10 unique and no growth, trigger a reset. No more infinite circles!

🔧 Key Functions Explained

getValidMoves(head)

Returns adjacent cells (up/down/left/right) that aren’t walls or body. Used by every AI.

aStar(start, goal)

Classic A* search. Maintains g, h, f scores and returns the shortest path or null.

isSafe(path)

Simulates following path, grows the snake, and checks tail reachability via BFS/A*—ensures survival after an apple.

isStuckInLoop()

Compares recent head positions to detect stalls. If trapped, we restart.

restart()

Clears the game interval, resets snake & apple, and starts fresh after a brief pause.

🧪 Lessons Learned

  • AI = decision rules, not magic
  • A* must be tempered with hazard checks
  • Planning + fallback outperforms raw speed
  • Loop detection saves your snake (and sanity)
